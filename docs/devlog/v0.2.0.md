## 技术要点

### 1. 棋盘二维数组
用 `self.board[col][row] = piece` 这样的二维数组存棋盘上的棋子，查找某个位置有没有棋子是O(1)，比遍历整个pieces列表快很多。
类比：JS里的二维数组，或者Java的二维数组。

### 2. 初始布局
中国象棋初始布局很标准：
- 底线：车马象士将士象马车
- 炮在第二行的2路和8路
- 兵在第三行的1、3、5、7、9路
红黑双方对称，红方在下面（行号大的一侧），黑方在上面。

### 3. 棋子层级
绘制的时候先画没选中的棋子，再画选中的棋子，这样选中的棋子一定在最上面，视觉上更清楚。
类比：CSS的z-index，选中的z-index更高。

### 4. 移动动画
用 `ease out` 缓动（`1 - (1-t)^2`），开始快结尾慢，感觉很自然。
移动的过程中棋盘坐标还是老的，动画结束才更新，这样逻辑和渲染分离。

## 踩过的坑

### 坑1：棋子位置和棋盘坐标对应
一开始棋子的位置是直接用`board_to_screen`算的，但移动动画的时候需要插值，所以加了`self.x`和`self.y`存屏幕坐标，update的时候根据棋盘坐标或者动画进度更新。

### 坑2：棋盘二维数组的索引
Lua的数组默认是1开始的，但我们棋盘坐标用的是0-8和0-9，所以要注意，初始化的时候要从0开始循环，不然会错位。
**解决：** 初始化时 `for col = 0, 8 do` 明确写0开始。

### 坑3：吃子的时候要从两个地方移除
吃子不仅要从pieces列表里移除，还要从board二维数组里移除，不然`get_piece_at`还能拿到，就bug了。
**解决：** `remove_piece`方法统一处理两个地方，外面不要直接改。
