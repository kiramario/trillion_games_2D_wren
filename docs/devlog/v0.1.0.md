## 技术要点

### 1. 棋盘坐标系统
- 棋盘用 (col, row) 表示，col 0-8（列，从左到右），row 0-9（行，从上到下）
- row 0-4 是黑方区域，row 5-9 是红方区域，中间是楚河汉界
- 坐标转换都封装在Board类里，外部不用关心像素位置

### 2. 相机系统
- 用 love.graphics.push/pop 做变换栈
- 变换顺序：平移到中心 → 缩放 → 旋转 → 平移相机位置
- 震动效果用随机偏移 + 衰减，很自然
- 屏幕/世界坐标转换跟绘制变换完全对应

### 3. 视觉效果
- 棋盘背景加了4像素的偏移阴影，营造一点厚度感
- 楚河汉界用浅蓝半透明横条，有河流的感觉
- 炮位和兵位有小十字标记，跟真实象棋盘一样
- 九宫格斜线也画了

## 踩过的坑

### 坑1：棋盘格子的尺寸计算
一开始以为是格子数是9x10，实际上是交叉点9x10，格子数是8x9。
**解决：** width = (COLS-1) * cell_size，height = (ROWS-1) * cell_size

### 坑2：楚河汉界的竖线处理
真实象棋盘中间的竖线是断开的，只有最左和最右两条是通的。
**解决：** 中间的竖线分上下两段画，两端的竖线整条画。

### 坑3：相机变换顺序
变换顺序很重要，顺序错了坐标就不对。
**解决：** 按照标准的2D相机变换顺序：平移到中心 → 缩放 → 旋转 → 平移相机。
