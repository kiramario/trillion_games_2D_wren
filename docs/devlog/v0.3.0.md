# 开发日志 - V0.3.0 走子吃子版

**版本号：** v0.3.0

## 本版本目标
完整的棋子走法校验，选中棋子后显示可走位置，可以正常走子和吃子，回合制。

## 完成的功能

### 游戏层新增
1. **Rules 走法规则模块**
   - 7种棋子完整走法校验：
     - 将/帅：九宫格内走一步，上下左右
     - 士/仕：九宫格内走一步斜线
     - 象/相：走田字，不能过河，塞象眼判断
     - 马：走日字，蹩马腿判断
     - 车：直线任意步，中间不能有子
     - 炮：空走跟车一样，吃子必须隔一个炮架
     - 兵/卒：没过河只能往前走，过河可左右走，不能后退
   - `is_valid_move()` - 检查某个走法是否合法
   - `get_valid_moves()` - 获取某个棋子所有合法走法
   - `is_in_check()` - 检查是否被将军（基础版）

### 游戏状态更新
- `move_history` 记录每一步走子历史
- 移动棋子带动画（缓动效果）
- 吃子逻辑正常工作

### 游戏场景更新
- 选中棋子后，高亮所有可走位置：
  - 空位置：绿色小圆点
  - 可吃的棋子：红色大圆圈
- 只能选中当前回合的棋子
- 点击可走位置就走子，不合法的走不了
- UI显示走子步数
- 走子自动切换回合

## 技术要点

### 1. 规则集中管理
所有走法规则都放在 `rules.lua` 里，跟游戏状态分开，这样：
- 规则改的时候不用动游戏状态的代码
- 后面做AI的时候可以直接复用规则模块
- 单元测试也方便（虽然V0阶段还没写测试）

类比：Java里的Service层，专门处理业务逻辑，跟数据模型分开。

### 2. 暴力遍历找可走位置
象棋棋盘只有9x10=90个位置，直接遍历所有位置检查是否合法，完全没问题。
不用做什么复杂的算法，简单清晰最重要。

### 3. 棋子移动动画
用 `ease out` 缓动，`self.move_time` 记录时间，`move_duration` 控制时长。
动画过程中，棋子的逻辑坐标（col, row）还是旧的，等动画结束才更新，这样逻辑和渲染分离，不会出bug。

### 4. 回合制
每次走子后自动切换 `current_turn`，选棋子的时候只能选己方的，保证不会乱走。

## 踩过的坑

### 坑1：象的走法，过河判断反了
一开始写反了，红方象可以过河，黑方的不行。
**解决：** 仔细对了一下，红方在下面（row大的一侧），象不能过河就是row不能小于5；黑方在上面，row不能大于4。

### 坑2：马的蹩马腿位置算错
一开始马腿的位置搞反了，横着走日字的时候马腿在竖方向，不对。
**解决：** dx=2 dy=1的时候是横着走两格，马腿在横方向中间；dx=1 dy=2的时候是竖着走两格，马腿在竖方向中间。

### 坑3：炮的吃子规则
炮空走和吃子规则不一样，一开始忘了区分，结果炮可以直接跳过去吃子。
**解决：** 先看目标位置有没有棋子，有的话就是吃子，中间必须恰好1个；没的话就是空走，中间必须0个。

### 坑4：兵的前进方向
红方和黑方的前进方向是反的，一开始都写成row减小了，黑兵只能往上走。
**解决：** 红方往前是row减小，黑方往前是row增大。
