# 开发日志 - V0.0.1 脚手架奠基版

**版本号：** v0.0.1  
**发布日期：** 2026-06-09  
**状态：** 完成

---

## 本版本目标
搭建通用2D游戏脚手架的基础框架，跟具体游戏无关，为后续开发打地基。

## 完成的功能

### 核心层 (Core)
1. **Logger 日志系统**
   - 支持 DEBUG/INFO/WARN/ERROR 四个级别
   - 不同级别不同颜色显示
   - 错误日志自动打印堆栈信息
   - 可配置日志级别

2. **Config 配置系统**
   - 集中管理所有配置项，分模块组织
   - 支持点路径访问（如 "graphics.background_color"）
   - 有默认值，可动态修改
   - 支持 dump 所有配置

3. **EventBus 事件总线**
   - 发布订阅模式，模块之间解耦通信
   - 支持订阅、取消订阅、发布事件
   - 回调执行有pcall保护，一个出错不影响其他
   - 可清除事件，可查看订阅者数量

4. **ResourceManager 资源管理器**
   - 统一管理图片、音效、字体资源
   - 自动缓存，避免重复加载
   - 支持卸载资源
   - 加载失败有错误处理，不崩溃

5. **Utils 工具函数**
   - 深拷贝/浅拷贝
   - 数组操作、table操作
   - 数学工具（clamp, lerp, round）
   - 字符串工具（分割、前缀后缀判断）
   - 几何工具（点是否在矩形内、距离计算）
   - ID生成、调试打印table

### 系统层 (Systems)
1. **RenderSystem 渲染系统**
   - 分层渲染（背景/游戏/特效/UI/调试五层）
   - 按层级顺序绘制，上层盖下层
   - 每帧清空，重新添加绘制项
   - 绘制函数有pcall保护

2. **InputSystem 输入系统**
   - 统一封装鼠标和键盘输入
   - 通过事件总线发布输入事件
   - 支持全局事件和具体按键事件
   - 可查询按键状态
   - 不破坏LÖVE2D原有的回调

### 场景层 (Scenes)
1. **BaseScene 场景基类**
   - 定义统一的生命周期接口
   - enter/exit/update/draw/pause/resume/resize
   - 类比面向对象的抽象基类

2. **SceneManager 场景管理器**
   - 场景注册、切换
   - 管理场景生命周期
   - 切换场景时触发事件
   - 支持传参

3. **MenuScene 菜单场景**
   - 测试用菜单，有三个选项
   - 支持键盘上下选择、回车确认
   - 演示分层渲染的使用
   - 按1键切换到游戏场景

4. **GameScene 游戏场景**
   - 测试用游戏场景
   - 演示鼠标输入处理
   - 点击鼠标画点，有淡出动画
   - 有棋盘占位图，为下一版本做准备
   - 按2键返回菜单

### 入口
- **main.lua**：游戏主入口，初始化所有系统，启动场景
- **conf.lua**：LÖVE2D配置，窗口、模块等

## 技术要点

### 1. Lua 面向对象实现
用 metatable + __index 模拟类和继承，统一写法：
```lua
local Base = {}
Base.__index = Base

function Base.new()
    local self = setmetatable({}, Base)
    return self
end

-- 继承
local Child = {}
Child.__index = Child
setmetatable(Child, {__index = Base})

function Child.new()
    local self = Base.new()
    setmetatable(self, Child)
    return self
end
```

### 2. 事件驱动解耦
模块之间不直接调用，通过EventBus通信。
好处：加新功能不用改旧代码，耦合低。

### 3. 分层渲染
所有绘制都通过RenderSystem，按层画。不用自己记绘制顺序，不会出现"这个怎么跑到上面去了"的问题。

### 4. 错误处理
重要地方都有pcall保护，一个地方出错不会整个游戏崩掉。
错误会打日志，方便排查。

## 踩过的坑

### 坑1：LÖVE2D的入口文件位置
LÖVE2D默认在运行目录找main.lua和conf.lua，不能放在子目录里。
**解决：** 把main.lua和conf.lua放在根目录，其他代码放src/，通过package.path设置搜索路径。

### 坑2：Lua的module搜索路径
默认只搜当前目录，所以src里的模块找不到。
**解决：** 在main.lua开头加 `package.path = 'src/?.lua;' .. package.path`

### 坑3：全局变量污染
Lua默认变量都是全局的，很容易污染。
**解决：** 尽量用local，核心模块才设成全局的，而且数量控制住。
