# 08. 发布路径建议

## 总览

目标平台优先级：
1. **Steam（桌面）**：最高优先级，首发平台
2. **Windows/macOS/Linux 桌面版**：和 Steam 一起做，本质差不多
3. **Android APK**：后续再做，优先级稍低

LÖVE2D 的好处是：**一次开发，多平台打包**。代码不用改，只要换打包方式就能出各个平台的包。

---

## 一、桌面版（Windows / macOS / Linux）

### 打包原理

LÖVE2D 游戏的本质是：
1. 把所有代码和资源打包成一个 `.love` 文件（其实就是 zip 压缩包，改后缀名）
2. 把 `.love` 文件和 LÖVE2D 的运行时合并到一起，就是可执行文件

不同平台的运行时不一样，所以要分别打包。

### 打包工具

推荐用官方的打包方式，或者用现成的工具：
- **love-release**：官方推荐的打包脚本，支持多平台
- **手动打包**：最基础，自己合并文件
- **love2d-build**：第三方打包工具

推荐先用手动打包，理解原理，后面再用自动化脚本。

### Windows 打包步骤

1. 下载 Windows 版本的 LÖVE2D 运行时（32位和64位都可以，推荐64位）
2. 把游戏打包成 `game.love` 文件
3. 把 `game.love` 和 `love.exe` 合并：
   ```bash
   copy /b love.exe+game.love game.exe
   ```
   或者 Linux/Mac 下：
   ```bash
   cat love game.love > game.exe
   ```
4. 把 `game.exe` 和所有 dll 文件放在一个目录里
5. 压缩成 zip 包，就是 Windows 版本

### macOS 打包步骤

1. 下载 macOS 版本的 LÖVE2D（.app 文件）
2. 右键显示包内容
3. 把 `game.love` 放到 `Contents/Resources/` 目录里
4. 修改 `Info.plist` 里的游戏名、版本号、图标等
5. 导出 .app，压缩成 .dmg 或者 zip 包

### Linux 打包步骤

1. 下载 Linux 版本的 LÖVE2D
2. 可以用 AppImage 方式打包，用户下载就能运行
3. 或者分 deb、rpm 等包，比较麻烦，推荐 AppImage
4. 或者直接提供 .love 文件，让用户自己装 LÖVE2D 运行

### 桌面版发布注意事项

1. **图标**：各个平台都要做对应尺寸的图标
2. **窗口标题**：conf.lua 里要设置正确的游戏名
3. **版本号**：统一管理，不要各个平台版本号不一样
4. **文件关联**：可选，让 .love 文件用你的游戏打开
5. **存档路径**：LÖVE2D 会自动存在用户目录下，不用管
6. **64位优先**：现在基本都是64位系统了，32位可以不做

## 二、Steam 发布

### 前提条件

1. 注册 Steam 开发者账号，交 100 美元的上架费（每个游戏）
2. 同意 Steam 的各种协议
3. 准备好商店页面素材（截图、宣传片、描述等）

### 发布流程

1. **在 Steamworks 新建游戏**
   - 填游戏基本信息
   - 设置商店页面
   - 上传商店素材

2. **打包游戏**
   - Windows 版本（必选）
   - macOS 版本（推荐）
   - Linux 版本（可选，看用户需求）

3. **上传到 Steam**
   - 用 SteamPipe 工具上传构建版本
   - 设置 Depot（每个平台一个 Depot）
   - 设置默认安装包

4. **设置 Steam 功能（可选）**
   - 成就
   - 云存档（LÖVE2D 的存档在用户目录，可以配置云同步路径）
   - 排行榜
   - 创意工坊（如果有必要的话）
   - 手柄支持（我们的输入管理器可以加手柄支持）

5. **审核与发布**
   - 提交商店页面审核
   - 提交构建审核
   - 设置发布日期
   - 正式发布

### Steam 集成

可以用 `love2d-steamworks` 库来集成 Steam 功能：
- 统计和成就
- 云存档
- 好友列表
- 联机（如果做联机的话）

如果不需要这些功能，也可以不集成，直接当普通游戏上架。

### Steam 发布注意事项

1. **游戏名**：不要和已有的游戏重名，提前搜一下
2. **商店页素材**：至少 5 张截图，一个宣传片，各种尺寸的图标和头图
3. **年龄分级**：中国象棋一般是全年龄，问题不大
4. **定价**：独立小游戏一般几美元到十几美元，自己定
5. **测试**：发布前一定要在不同的 Windows 版本上测一下，别运行不起来
6. **中文支持**：游戏本身是中文的，商店页也做中文的

## 三、Android APK 打包

### 难度说明

Android 打包比桌面版麻烦一点，但也是可行的。LÖVE2D 官方支持 Android。

### 打包方式

方式一：用官方的 Android 构建模板自己编译
- 需要 Android SDK、NDK
- 比较麻烦，但最灵活

方式二：用现成的打包工具
- `love-android`：官方的 Android 移植版
- 有现成的 APK，把游戏塞进去就行
- 或者自己编译

方式三：用第三方打包服务
- 有些在线服务可以把 LÖVE2D 游戏打包成 APK
- 简单，但可能收费或者有限制

推荐：先用方式二快速验证，后面再优化成方式一。

### 打包步骤（简化版）

1. 下载 LÖVE2D 的 Android 模板 APK
2. 把 `game.love` 文件放到 APK 的 assets 目录里
3. 修改包名、应用名、图标
4. 重新签名 APK
5. 测试安装运行

### 移动端适配注意事项

1. **屏幕适配**：手机屏幕分辨率各种各样，要做自适应
2. **触控操作**：我们的游戏现在是鼠标操作，手机上要改成触控，其实差不多，点击对应按下
3. **性能**：手机性能比电脑差，粒子效果、动画不要太夸张
4. **内存**：资源不要太大
5. **存档**：Android 上存档也是存在应用私有目录里
6. **横竖屏**：竖屏还是横屏？象棋竖屏也可以，横屏也可以，看设计

### 上架应用商店

如果要上架国内应用商店，比较麻烦：
- 需要软著
- 需要各种备案
- 每个商店都要单独上架
- 可能需要支付 SDK 之类的

如果只是自己玩或者发 APK 给朋友，就不用上架，直接发 APK 文件就行。

## 四、其他平台（可选）

### iOS
- 也可以打包，但需要开发者账号（99美元/年）
- 比 Android 更麻烦
- 优先级最低，可以先不考虑

### Web 版（HTML5）
- LÖVE2D 有 Web 移植版（love.js）
- 可以把游戏打包成网页版
- 性能会差一点，但简单的游戏没问题
- 适合做试玩版

### Switch / 主机
- 也有办法移植，但门槛高，需要开发者资质
- 暂时不用考虑

## 五、版本号管理

统一用语义化版本号（Semantic Versioning）：
- 格式：v主版本号.次版本号.修订号
- 主版本号：大改动，不兼容的修改
- 次版本号：新增功能，向下兼容
- 修订号：Bug 修复，向下兼容

示例：
- v0.1.0：第一个可运行版本
- v1.0.0：正式发布第一个版本
- v1.0.1：修复 Bug
- v1.1.0：加新功能
- v2.0.0：大版本更新，可能不兼容

## 六、发布 Checklist

每次发布前检查：

### 功能检查
- [ ] 所有核心功能正常
- [ ] 没有严重 Bug
- [ ] 游戏可以完整玩到结束
- [ ] 存档读档正常
- [ ] 设置保存正常

### 资源检查
- [ ] 所有资源都打包进去了
- [ ] 没有多余的测试资源
- [ ] 图片音频都压缩过，不要太大
- [ ] 图标都换好了

### 配置检查
- [ ] 游戏版本号正确
- [ ] 窗口标题正确
- [ ] 调试模式关掉了
- [ ] 日志级别改到 WARN 或者 ERROR（不要输出太多调试日志）

### 测试
- [ ] 在 Windows 上测试过
- [ ] 在 macOS 上测试过（如果发 macOS 版）
- [ ] 在不同分辨率下测试过
- [ ] 存档目录清理后重新测试
- [ ] 从头玩一遍通关

## 七、推荐工具和资源

### 打包工具
- **love-release**：官方推荐的多平台打包脚本

### 图标制作
- 可以用 GIMP 或者在线工具生成各种尺寸的图标
- 各个平台需要的尺寸不一样，提前准备好

## 八、发布路径建议（按优先级）

### 第一阶段：桌面发布（优先）
1. 先打包 Windows 版本，自己测试没问题
2. 再打包 macOS 和 Linux 版本
3. 直接发布到 itch.io 之类的独立游戏平台，或者自己发
4. 同时准备 Steam 上架的材料

### 第二阶段：Steam 发布
1. 注册 Steam 开发者账号
2. 准备商店页素材
3. 上传构建版本
4. 商店页审核
5. 正式发布
6. 根据反馈更新版本

### 第三阶段：移动端（可选）
1. 先打包 Android APK，自己测试
2. 适配触控操作和屏幕
3. 优化性能
4. 考虑要不要上架应用商店

### 暂时不用管的
- iOS、Switch、Web 版这些，等后面有精力再做

## 九、常见坑点

### 桌面版
1. **Windows 上缺少 dll**：要把所有 dll 都带上，不要只发 exe
2. **macOS 上提示损坏**：因为没签名，用户需要右键打开，或者去设置里允许
3. **存档路径问题**：不同系统不一样，用 love.filesystem 就没问题，不要自己写绝对路径

### Steam
1. **审核时间**：商店页审核可能要几天，提前准备
2. **云存档配置**：要正确设置同步路径，不然会丢档
3. **成就统计**：要提前设计好，上线后再加比较麻烦

### Android
1. **性能问题**：手机性能比电脑差，要优化
2. **触控延迟**：尽量减少输入延迟
3. **内存限制**：大图片、大音效用流式加载
4. **签名问题**：一定要保存好签名文件，丢了就不能更新了

---

## 总结

LÖVE2D 的跨平台能力还是很强的，一次开发多平台发布不是空话。

**建议顺序：先做好桌面版，上 Steam，后面再考虑移动端。**

先把游戏本身做好，发布的事情后面自然水到渠成。
