## 二、桌面版发布（Windows/macOS/Linux）

### 2.1 基础原理
LÖVE2D 游戏的本质是一个 `.love` 文件（就是zip压缩包改后缀名） + LÖVE2D 运行时。
打包就是把你的游戏代码和运行时合并到一起，用户不用单独安装LÖVE2D就能玩。

### 2.2 第一步：生成 .love 文件
```bash
# 在项目根目录执行
cd src
zip -r ../build/game.love .
# 注意：conf.lua 和 main.lua 必须在zip的根目录，不能在子目录里
```

验证：
```bash
love build/game.love
# 能正常运行就对了
```

### 2.3 第二步：合并到 LÖVE2D 可执行文件

#### Linux 版
```bash
# 下载官方的Linux版LÖVE2D
cat love game.love > game
chmod +x game
```

#### Windows 版
```bash
# 下载官方的Windows版LÖVE2D（32位或64位）
# 解压后得到 love.exe 和一堆dll
copy /b love.exe+game.love game.exe
```
然后把 game.exe 和所有dll、license文件放在一起，就是完整的游戏了。

#### macOS 版
- 下载官方的 love.app
- 右键显示包内容
- 把 game.love 放到 Contents/Resources/ 里
- 改一下 app 的名字、图标、Info.plist

### 2.4 第三步：打包成安装包（可选）
- Windows：用 Inno Setup 或 NSIS 做成安装程序
- macOS：用 hdiutil 做成 .dmg 镜像
- Linux：做成 AppImage，或者 deb/rpm 包

### 2.5 工具推荐：love-release
可以用 love-release 这个工具自动打包多平台：
```bash
# 安装（需要Lua和LuaRocks）
luarocks install love-release

# 使用
love-release -W  # Windows
love-release -M  # macOS
love-release -L  # Linux
```

## 三、Steam 发布指南

### 3.1 前置准备
1. **Steam 开发者账号**：需要交100美元（约700人民币），每个游戏
2. **游戏信息准备**：
   - 游戏名称
   - 游戏描述（中文+英文）
   - 宣传图、截图、预告片
   - 系统需求
   - 年龄分级
3. **法律相关**：隐私政策、EULA、用户协议等

### 3.2 技术接入

#### Steamworks SDK 接入
要使用Steam的成就、排行榜、云存档等功能，需要接入Steamworks SDK。
Lua版的Steamworks绑定：**luasteam**

接入步骤：
1. 下载 luasteam
2. 把编译好的 .so/.dll 放到游戏目录
3. 代码里 require("luasteam")
4. 初始化、处理回调等

**注意：**
- V0阶段不用接，先把游戏做好
- 到V1.0的时候再接也不晚
- 不接Steamworks也能在Steam上发，只是没有Steam的那些功能

### 3.3 上传到 Steam
1. 登录 Steamworks 后台
2. 创建新的 App ID
3. 填写商店页面信息
4. 用 SteamPipe 上传构建版本
   - 可以用官方的 SteamCMD 命令行工具
   - 也可以用图形界面的 Steamworks SDK 里的工具
5. 设置好 depots（不同平台的文件）
6. 提交审核

### 3.4 审核与发布
- 审核一般1-3天
- 第一次发可能会慢一点
- 棋牌类游戏一般没问题，没有违规内容都能过
- 通过后就可以设置发售日期，正式发布了

### 3.5 注意事项
1. **先弄商店页面**：商店页面审核也要时间，可以先弄商店页面，游戏还没做好也能先放个Coming Soon页面攒愿望单
2. **云存档**：做了存档功能的话，建议接Steam云存档，体验好
3. **成就**：可以加几个简单的成就，增加完整体验
4. **测试**：用Steam的Beta分支，可以发测试版给玩家测

## 四、Android APK 发布指南

### 4.1 基本原理
LÖVE2D 有官方的 Android 移植版，把 .love 文件打包进APK里就能运行。

### 4.2 方法一：用官方模板（推荐）
1. 下载 LÖVE for Android 的源码或预编译模板
2. 把你的 game.love 放到 assets/ 目录
3. 用 Android Studio 打开项目
4. 修改包名、应用名、图标等
5. 编译生成 APK

### 4.3 需要适配的地方
1. **输入适配**：
   - 手机是触屏，没有鼠标
   - 点击事件跟鼠标点击差不多，基本不用改太多
   - 没有键盘，菜单什么的要都能点
2. **屏幕适配**：
   - 手机屏幕比例各种各样，要做自适应
   - 横竖屏问题：象棋建议竖屏还是横屏？象棋建议横屏，棋盘宽
3. **性能**：
   - 手机性能比电脑差，粒子特效什么的要注意优化
4. **权限**：
   - 不要申请不必要的权限，审核容易过

### 4.4 发布到应用商店
- Google Play：需要开发者账号（25美元）
- 国内应用商店：各个渠道都要单独上架，比较麻烦
- 建议：如果不是特别想做移动端，就先不做，先做好PC版

### 4.5 注意事项
- Android 版本的 LÖVE2D 可能跟桌面版有一些API差异，要测试
- 触屏操作手感要调，跟鼠标不一样
- 字体在手机上可能显示不一样，要测试

## 五、发布时间线建议

### 阶段1：开发期（V0 - V0.9）
- 不用管发布的事
- 专心做游戏
- 用桌面版开发调试

### 阶段2：接近完成（V0.9 - V1.0）
- 注册Steam开发者账号
- 开始弄商店页面
- 准备宣传素材
- 接Steamworks（如果要接的话）

### 阶段3：正式发布（V1.0）
- 桌面版打包测试
- 上传Steam审核
- 同时可以发itch.io测一下
- Steam审核通过后正式发售

### 阶段4：后续更新
- 发完PC版稳定了，再考虑移动端
- 或者先不做移动端，专心更PC版

## 六、常见坑

### 坑1：路径大小写问题
- Windows 不区分大小写，Linux/macOS/Android 区分
- 所有文件名、路径名都用小写，避免踩坑
- require 的路径也要注意大小写

### 坑2：资源路径
- 打包后资源路径跟开发时可能不一样
- 用相对路径，不要用绝对路径
- ResourceManager 统一处理，不要到处拼路径

### 坑3：版本兼容
- 开发用的LÖVE2D版本和打包用的版本要一致
- 建议统一用 11.5 版本，目前最稳定的稳定版

### 坑4：中文显示
- LÖVE2D默认字体不支持中文
- 一定要加载中文字体，不然中文全是方块
- 字体文件要注意版权，用免费可商用的

### 坑5：Steam 反作弊
- 纯单机游戏不用管
- 不要随便加奇怪的dll，不然可能被当成挂

## 七、替代发布渠道
如果Steam审核不想等或者过不了，可以发这些：
1. **itch.io**：独立游戏平台，上传简单，没有审核，抽成10%（你自己定）
2. **GameJolt**：类似itch.io
3. **自己官网卖**：用PayPal或支付宝，自己收款，100%到手
4. **Epic Games Store**：抽成12%，但审核更严，小游戏不一定能上

**建议：**
先上itch.io测试，同时弄Steam，Steam过了主要卖Steam，itch.io作为补充。
