# 01. 项目愿景

## 项目定位

这是一个独立开发者的 2D 游戏项目，以**中国象棋**作为首发作品，目标是先上线 Steam，后续扩展到移动端（Android APK）和多桌面平台（Windows/macOS/Linux）。

但更重要的是，这不仅仅是一个象棋游戏，而是一套**可复用的 2D 游戏基础框架**。后续可以基于这套框架快速开发其他 2D 游戏，例如弹珠、卡牌、休闲益智类游戏，不需要重复造轮子。

## 核心目标

### 短期目标（0-3 个月）
1. 完成中国象棋的完整可玩版本，上线 Steam
2. 打磨基础框架，确保核心模块稳定、通用、可复用
3. 建立完整的开发、测试、发布流程

### 中期目标（3-6 个月）
1. 推出 2-3 款基于同一框架的小游戏，验证框架复用性
2. 支持移动端打包发布（Android APK）
3. 建立美术资源 pipeline 和 UI 组件库

### 长期目标（6-12 个月）
1. 形成一套成熟的 2D 独立游戏快速开发框架
2. 可以在 1-2 周内从 0 到 1 做出一款可上线的休闲游戏
3. 持续积累通用模块和工具链

## 设计理念

### 1. "2D 但有纵深感"
- 不做真 3D（避免晕 3D）
- 通过分层渲染、光影、透视错觉、粒子特效、镜头运动营造空间感
- 借鉴成熟产品的交互反馈和节奏，但不抄袭具体内容

### 2. 完全可控
- 代码结构、模块边界、版本节奏全部自己掌控
- 不使用黑箱式的游戏引擎插件，所有核心逻辑自己实现
- 代码简洁、注释充分，方便后续维护和扩展

### 3. 通用优先
- 核心框架层（src/core/）和游戏业务层（src/game/）严格分离
- 核心层不依赖任何特定游戏的逻辑
- 每个通用模块都考虑复用性，不是只为象棋服务

### 4. 渐进式开发
- 每个版本都是最小可运行版本
- 随时可以停下来玩，都能看到明确成果
- 不追求一步到位，逐步迭代完善

## 目标用户

### 首发游戏（中国象棋）
- 喜欢传统棋类游戏的玩家
- Steam 平台的休闲游戏用户
- 移动端的象棋爱好者

### 框架本身
- 自己（后续开发其他游戏时复用）
- 可能的其他独立开发者（如果未来开源的话）

## 成功标准

1. **中国象棋上线 Steam**，有基础的可玩性和完整的游戏流程
2. **框架可以快速复用到第二个游戏**，验证通用层的设计是成功的
3. **开发效率提升**：做第二款游戏的时间比第一款短 50% 以上
4. **代码可维护**：时隔几个月回来，依然能快速看懂和修改
