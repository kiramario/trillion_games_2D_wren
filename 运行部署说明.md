## 本地运行

### 方式一：Linux 绿色版（推荐，本项目默认）
我们已经准备好了绿色版LÖVE2D，不需要安装，直接用：

```bash
# 进入项目目录
cd ~/love2DGame_wren

# 直接启动
./run.sh
```

如果提示找不到LÖVE2D，执行以下命令安装绿色版：
```bash
mkdir -p ~/apps && cd ~/apps
wget https://github.com/love2d/love/releases/download/11.5/love-11.5-x86_64.AppImage
chmod +x love-11.5-x86_64.AppImage
./love-11.5-x86_64.AppImage --appimage-extract
# 解压后会生成 squashfs-root 目录，就是绿色版
```

### 方式二：系统安装版
如果你已经系统安装了LÖVE2D：

```bash
cd ~/love2DGame_wren
love .
```

### 方式三：Windows / macOS
1. 去官网下载安装LÖVE2D：https://love2d.org/
2. 把项目文件夹拖到 love.exe / love.app 图标上就能运行
3. 或者命令行：`love.exe 项目路径`

## 打包发布方法

### 1. 桌面版打包（Windows / macOS / Linux）

#### Linux → AppImage
```bash
# 把游戏打成 .love 文件
cd ~/love2DGame_wren
zip -r game.love . -x "*.git/*" "docs/*" ".gitignore"

# 跟LÖVE2D合并成单独的可执行文件
cat ~/apps/squashfs-root/usr/bin/love game.love > game
chmod +x game
```

#### Windows
1. 下载Windows版的LÖVE2D
2. 把 game.love 和 love.exe 合并：
   ```cmd
   copy /b love.exe+game.love game.exe
   ```
3. 把DLL和资源文件跟game.exe放一起
4. 用 Inno Setup 做成安装包

#### macOS
1. 下载macOS版的love.app
2. 右键显示包内容
3. 把 game.love 放到 Contents/Resources/ 里
4. 改 Info.plist 里的包名、图标、版本号
5. 可选：签名、公证

### 2. Steam 发布
1. 先打好桌面版包
2. 注册Steam开发者账号，交100美元
3. 创建游戏应用，填写商店信息
4. 用 Steamworks SDK 对接：
   - 成就
   - 排行榜
   - 云存档
   - 可以用 love2d-steamworks 库（Lua封装好的）
5. 上传构建到Steam后台
6. 审核通过后发布

### 3. Android APK 打包
1. 用 love-android 项目：https://github.com/love2d/love-android
2. 把 game.love 放到 assets 目录
3. 修改配置：包名、图标、权限
4. 用 Android Studio 编译出APK
5. 签名、对齐
6. 上架Google Play或国内应用商店

**注意：** APK需要适配触摸操作，现在版本是鼠标操作，手机上需要改一下输入处理。

## 常见问题

### Q1：启动报错说找不到love？
A：LÖVE2D没装或者路径不对。按照run.sh里的提示安装绿色版，或者系统安装LÖVE2D。

### Q2：中文显示成方框？
A：LÖVE2D默认字体不支持中文，我们代码里动态生成的字体其实是支持中文的，如果还有问题可以换个中文字体文件放到assets/fonts目录下。

### Q3：AI难度4特别卡？
A：专家难度是4层搜索，局面复杂的时候可能要想几秒到十几秒，正常现象。一般玩困难（3层）就够了，速度和强度平衡比较好。

### Q4：怎么切换回双人对战？
A：按 M 键就能切换人机/双人模式，切换后会自动重开。

### Q5：悔棋怎么只悔一步？
A：人机模式下默认悔两步，把AI的一步也悔掉，直接回到你走之前的状态。双人模式只悔一步。如果你想一步一步悔，可以改代码里的悔棋逻辑。

### Q6：窗口大小可以改吗？
A：可以，窗口支持缩放，棋盘会自动适配。也可以改 src/core/config.lua 里的默认窗口大小。

### Q7：代码怎么改？
A：Lua是脚本语言，改完保存直接重新运行就行，不用编译。非常方便快速迭代。

## 注意事项

### 开发注意
1. **通用层和业务层分开**：
   - core/、systems/、scenes/ 这几层是通用的，跟象棋无关
   - entities/、game/ 是象棋业务相关的
   - 后面做别的游戏（比如弹珠），只需要替换entities和game两层，下面的直接复用
   - 改代码的时候注意不要把游戏业务写到通用层里

2. **Lua语法小提示**：
   - 数组默认从1开始，不是0！（我们的棋盘坐标特意做成0开始，是为了符合棋盘习惯，注意区分）
   - 变量默认是全局的，所以局部变量一定要加 `local`
   - 表（table）就是数组和对象的结合体，用起来跟JS的对象差不多
   - 没有类，但可以用setmetatable模拟，我们的代码里都是这么写的

3. **性能注意**：
   - LÖVE2D性能很好，做2D游戏完全够用
   - AI搜索深度别太深，4层就差不多了，再深普通电脑会卡
   - 不要在draw函数里做复杂计算，尽量放到update里

### 发布注意
1. **Steam优先**：你说目标优先Steam，那就先把桌面版做好，上Steam，移动端后面再说
2. **素材版权**：现在都是程序生成的图形，没有版权问题。后面换美术资源一定要注意版权，最好自己画或者买商用授权的
3. **版本号管理**：每次发版记得打tag，跟我们现在的做法一样，方便回滚
4. **存档格式**：后面做存档的话，用Lua的table序列化就行，简单方便
5. **测试**：每个版本都要多测测规则有没有bug，象棋规则很容易出边界case

### 架构扩展注意
如果后面要做别的2D游戏，比如弹珠台：
1. 新建 `src/game/pinball/` 目录放弹珠的业务代码
2. 新建 `src/entities/pinball/` 放弹珠的实体
3. core、systems、scenes 层不用动，直接复用
4. 可以做多游戏入口，菜单里选玩哪个游戏
5. 这样整个框架就真的变成通用2D游戏脚手架了
